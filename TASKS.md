- [x] **Performance & security audit + fixes (June 2026)** — Full codebase audit identified 6 issues. All fixed and deployed:
  1. **Leaderboard ISR broken** — `cookies()` call in `leaderboard/page.js` was silently opting the page out of the 24h cache (`revalidate=86400` was a no-op). Fixed by switching to the anon `supabase` client. IMPORTANT: any page that uses `revalidate` must NOT call `cookies()` or the cache is bypassed. `/leaderboard` is the only ISR page; all others are either SSG or force-dynamic.
  2. **`getCurrentSignals()` truncation bug** — query had no date filter; after ~8 months of `tactical_monthly_holdings` history the Supabase 1,000-row cap would silently drop signal portfolios later in the alphabet. Fixed with a 2-step query: fetch latest date first, then only that date's rows.
  3. **`/builder` cold-start weight** — `getAllAllocations()` + `getCurrentSignals()` were fetched server-side on every page open. Moved to `GET /api/builder-holdings?slugs=` fetched client-side only when 2+ portfolios are selected. `BuilderClient.jsx` no longer accepts `allAllocations`/`allSignals` props.
  4. **`/account` wasted allocation fetch** — `getAllAllocations()` fetched even for users with 0 saved mixes. Now gated on `savedMixes.length > 0`.
  5. **`builder-save` name length** — only client-side `maxLength={80}`; server now trims + slices to 80.
  6. **`auth/callback` next param** — `next` now validated to start with `/` and not `//` (defense-in-depth against open-redirect edge cases).

Root cause: Memberful does NOT fire any webhook when a member self-cancels through the member portal — it only fires `subscription.deactivated` when the billing period actually expires. So the `user_subscriptions` status column cannot be updated in real time on cancellation. Fix: account page now shows "Access until [date]" for all subscription statuses (removed "Renews" wording) to avoid confusion during the grace period. `app/api/memberful/route.js` also refactored: `subscription.updated` is now handled separately from `activated/created/renewed` — it detects cancellation via `changed.autorenew.new === false` and reactivation via `changed.autorenew.new === true` (for any future scenarios where Memberful does send that event), and never blindly overwrites status to 'active'. ## Competitive Gaps (from Allocate Smartly audit, June 2026)

Features Allocate Smartly has that PortfolioDB genuinely does not. Ordered by impact-to-effort ratio.

- [x] **Portfolio detail page — additional stats (% Profitable Months, Longest Drawdown, Best/Worst Month, Annualized Volatility)** — Four stats shown on every AS strategy page that PDB doesn't surface: (1) **% Profitable Months** — percentage of months with a positive return; (2) **Longest Drawdown Duration** — how many consecutive months the portfolio was below its prior peak (PDB shows max drawdown depth but not duration); (3) **Best Month / Worst Month** — single-month extremes (PDB shows Best/Worst Year); (4) **Annualized Volatility** — standard deviation of monthly returns × √12, shown as an explicit stat (currently implied by Sharpe but never displayed directly). All four are computable from existing `monthly_returns`. Add to `portfolio_stats` view, `lib/statDefinitions.js`, and the Performance Snapshot section on portfolio detail pages. Low effort.

- [x] **Portfolio detail page — benchmark side-by-side column on stats table** — AS displays every stat as two adjacent columns: "Portfolio / 60:40 Benchmark." PDB shows stats in isolation with no immediate calibration reference. A user seeing a 7.2% CAGR doesn't know if that's good without context. Implementation: add a second value column to the Performance Snapshot `StatRow` entries showing the same stat for the US 60/40 benchmark. Benchmark data already fetched server-side in `portfolios/[slug]/page.js`. Low-medium effort.

- [ ] **Portfolio detail page — annual returns table** — A scannable year-by-year grid showing each calendar year's return (e.g. 2008: -32.4%, 2009: +28.1%). AS shows this going back to 1970. PDB has the holding-period heatmap (start year × holding period) and charts, but no simple annual returns table. Users frequently want to look up "how did this do in [specific year]?" Data entirely computable from `monthly_returns`. Implementation: server-computed `annualReturns` array passed as prop, rendered as a compact table on the portfolio detail page. Color-coded green/red by positive/negative. Low effort.

- [ ] **Risk vs. Return scatterplot — "Portfolio Map" (`/tools/portfolio-map`)** — All 70 portfolios plotted on a single chart: annualized volatility on X-axis, annualized return on Y-axis. Instantly answers "where does this portfolio sit relative to everything else?" — a visual overview no other single page on the site provides. Hovering a dot shows the portfolio name and stats; clicking navigates to its detail page. Optional: color-code dots by category (Buy and Hold / Tactical / Robo-Advisor). No efficient frontier needed for our use case. All data computable from `portfolio_stats` (CAGR + annualized volatility once that stat is added — see above). Implementation: new server page + Recharts `ScatterChart` client component. Add to Tools dropdown and sitemap. Low-medium effort.

- [x] **Correlation matrix (`/tools/correlation`, June 2026 — redesigned same month, see CLAUDE.md for current layout details)** — Full pairwise correlation heatmap across all portfolios. `app/api/correlation-matrix/route.js` (GET, `revalidate = 86400`) fetches all `monthly_returns` via parallel `.range()` pagination, groups into `Map<date, return>` per slug, and computes Pearson correlation on each pair's overlapping months (null if < 24 months overlap or < 24 months total history). Returns `{ portfolios, matrix }`. `CorrelationMatrixClient.jsx` now renders inline numeric correlation values in uniform 48px cells (fully-vertical column headers via `writing-mode: vertical-rl`), a relative green→red color scale anchored to the dataset's actual min/max (not a sign-based diverging scale — this dataset has zero negative correlations so that rendered all-red), category filter pills, a "Select portfolios" multi-select picker that narrows the matrix to an exact column set while keeping the full list down the rows, and a search box that dims non-matching rows/columns by slug. Hover panel shows the pair's correlation value + plain-English label; click any off-diagonal cell to open `/compare?slugs=a,b`. Added to Tools dropdown (desktop + mobile), sitemap, and CLAUDE.md.

- [ ] **10-Year Market Valuation Context widget** — A simple gauge or trend chart showing current US stock market valuation using publicly available Shiller CAPE data (source: Shiller's Yale dataset, no API key required). Gives buy-and-hold users macro context for why forward returns might differ from historical averages — directly relevant to interpreting backtests. AS has a full ensemble model; a minimal version (CAPE trend line + current value + historical median reference line) would be low effort and meaningful. Implementation: fetch CAPE data server-side, cache in Supabase or as a static JSON, render on a new `/tools/market-valuation` page or as a contextual widget on portfolio detail pages. Low-medium effort for display-only version.

- [ ] **Underperformer Watchlist** — Flags portfolios whose recent performance has statistically underperformed their long-term backtest expectation, with a severity rating (1–3) and plain-English status note. Signals active curation and intellectual honesty — a strong trust signal for a data product. The main cost is methodology: needs a defensible definition of "expected" performance (rolling z-score vs. historical distribution is the natural approach), a monthly update process, and editorial descriptions for flagged entries. Implementation: new column in `portfolio_stats` view for rolling z-score; monthly update script; standalone `/tools/watchlist` page; badge on each portfolio detail page when flagged. High effort due to methodology design and ongoing curation. — The Material Symbols CSS `<link rel="stylesheet">` in `app/layout.tsx` is render-blocking, costing ~150ms on FCP (flagged by PageSpeed Insights, May 2026). Fix: change to `rel="preload" as="style"` + `onLoad="this.rel='stylesheet'"` pattern with a `<noscript>` fallback. Icons will be briefly unstyled (show as text characters) until the font loads — acceptable trade-off. Expected improvement: ~3–5 PageSpeed points.
